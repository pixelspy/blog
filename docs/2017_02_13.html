La seule constante est le changement

Article du Lundi 13 Février


Travail avec tout le groupe: création d'un mémo de toutes les balises HTML:
1) Lister les balises HTML sur post-its;
2) Répartition de la tâche, en suivant l'ordre alphabétique des balises;
3) Organisation des balises par famille;
4) Création de petits groupes;
5) Répartition des familles de balises aux groupes;
5) Décision commune de nommer 3 chefs de projet (qui changeront à chaque étape du projet);
6) Premier groupe de chefs (Rose, Daniel et Charles) créent la structure de notre Site Mémo




Révision de commandes Git Hub:
- Pour récupérer des modifs du dev Lead sur un dossier déjà forker et cloner:
1) git remote add upstream [url du dossier du lead]
2) git pull upstream master = pour récupérer les commits en local
- Pour récupérer un dossier github d'un autre dev sur mon remote GitHub et mon local, faire des modifs et les envoyer sur GitHub:
1) sur GitHub: FORK du compte de l'autre dev ou lead dev
2) creation d'un dossier en local (optionnel)
3) Ouvrir le terminal et : cd (dossier créé)
4) git innit
5) copier l'URL HTTPS sur mon GitHub
6) Terminal: git clone (url https)
7) Ouvrir le document souhaité, faire les modifs, travailler
8) Terminal: git add (fichier), git commit -m'message', git push origin master
9) sur GitHub : dans le dossier forké: faire un pull request
10) l'autre dev accepte le pull request.
